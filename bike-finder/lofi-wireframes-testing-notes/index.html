Tester1

Persona: Commuter John Johnson - As a person with back pain, I want to be able to find the right size and style of bike so that I don't injure my back further.

  • For the intro page, the questionaire section, providing an option for the user to close the questionaire is clear and user centered way to easily visit and exit.
  • The title and the information/questionaire that follows are not matching very well -> a bit confusing -> how about changing the title of the questionaire -> for example, "What bike should I get?" -> show what the end result would be.
  • The arrow indication for the size of bike that the user is looking for is not clear. Why would anyone not want the perfect size?
  • Indication of how many pages/questionaire there are is clear and a nice user centered UI.
  • On the main page where the bike image takes the whole page looks like a button (but could be because of current low-fidelity UI).
  • Since there is a "like" button in the navigation bar, how about thinking about adding like/heart icon on the products as well.
  • Because the commuter has no background knowledge about the bike, there is no need to use the filter function. -> It could be a good idea to think about beginner friendly section or filter function. For example, what terrain? help button? with which purpose matches with which kind of wheel?
  • In the product detail page, "change button" button next to the title need more clear naming.
  • Instead of "next step" button, how about changing its title to "add to cart."

Takeaways

After four different users' user testing, repeated feedbacks were unclear labels of the buttons. I tried to be concise in order to save space for other details of the design, and I believed that the users would be familiar with those terms so those labels would be easy to understand. However, because they were too concise and simple, it had not enough information for the users to understand. Moreover, navigation section and footer sections are vital in every page not only for the consistency but also for the users to easily navigate to other pages as well. Back buttons were missing in most of my pages, which can be solved if I put the navigation bar in every page. Lastly, because bike webiste requires many filters as there are many varieties of bikes, it is important to indicate which filters/details have been applied to the current product lists. This would help the users which filter is necessary and which is not. When building a high-fidelity wireframe, it is also important where specific details of the product should be placed and how much should be shown in the preview page.
